Themes

Mainframe supports themes to make it easier to implement a consistent interface in your application.
Do not think of themes in the sense of a wordpress theme where all presentation files are bundled in a single package. In Mainframe all presentation files are still closely related to each other, but you won't be able to distribute a theme in a zip file since themes in Mainframe are mostly a convention.

Themes are a way to organize your presentation layer files (views, css, js) in a way that makes it easy for you to make small incremental updates to your interface. This philosophy is also present in Mainframe's plugin system.

Loading themes

If you got Mainframe up and running, then you already have a theme running as well. Let's take a look at Mainframe's existing code and see how it's done.

First let's have a look at the default controller in /app/controllers/home.php.
The first line inside the index() method is: $this->load->theme('demo'); This tells Mainframe that it should use the demo theme to render any view used in this method.

How themes work

Everytime you load a view after loading a theme, Mainframe takes all the output from that view, and puts it in the place of the $content variable inside the theme.php file. Then it renders everything back to the browser.

Since the purpose of the theme is to be used repeatedly for all views, it is the one that calls all necessary CSS, Less, JavaScript and image files. These files are typicaly located for your convenience, in the corresponding folders inside /themes/theme_name/.

A view rendered within a theme



In order for the theme to be able to load your views correctly, the view files must be placed in a folder named after the theme.
For example: /app/views/theme_name/view.php.

However when loading a view you just use the view name, just like in a vanilla CodeIgniter installation: $this->load->view('view');

Loading assets in themes

Any files you load using the shorthands of the interface helper (CSS, LESS & JavaScript) will be placed
in the location specified by $this->load->assets()

Of course you can always overide the asset loading using normal HTML when you see fit.
